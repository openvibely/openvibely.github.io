Core Platform

Everything you need to run reviewable, project-aware agent workflows

From the first conversation to shipped code and reusable learning, OpenVibely keeps execution visible, steerable, and tied to your repository.

Task Board & Execution Workspace

Manage backlog, active, and completed work from a project-scoped board. Follow each task through streaming execution, threads, attachments, changed files, goals, and review.

  • Run, cancel, queue, and steer work in real time
  • Task threads preserve follow-up context
  • Changed files, diffs, and review comments
  • Task Goals continue unfinished objectives
  • Multi-role swarm execution

Smart Scheduling

Schedule one-time, recurring, and system maintenance work with a visual calendar and automatic dispatch when each run becomes due.

  • Calendar-based scheduling UI
  • One-time and recurring task runs
  • Automatic dispatch and run history

Model & Provider Flexibility

Configure Anthropic, OpenAI, Ollama, OpenAI-compatible providers, or Mixture of Models virtual configs with practical auth, defaults, tool policy, and capacity controls.

  • Built-in and custom compatible providers
  • OAuth, API key, and CLI auth options
  • Model defaults, tool policy, and routing

Reusable Agent Profiles

Define specialized workers with prompts, personalities, model settings, skills, plugins, MCP servers, permissions, routing hints, and lifecycle behavior.

  • Standalone and agent-owned skills
  • Scoped tools, plugins, and permissions
  • Lifecycle hooks before and after tasks

Git Worktrees & PR Flow

Run tasks in isolated git worktrees, inspect branch-level diffs and review comments, then merge or publish pull requests from task output.

  • Per-task git worktree isolation
  • Branch lineage, rebase, and merge controls
  • Pull requests and authorized review follow-ups

Channels, Alerts & API

Create, track, and report work through the web UI, Slack, Telegram, Discord, Email, GitHub, inbound webhooks, outbound targets, and the REST API.

  • Project-aware, authorized channel workflows
  • Failure, follow-up, and attention alerts
  • Streaming events and Swagger API docs

Compounding Intelligence

Every completed task can make the next one better

OpenVibely builds inspectable project memory and reusable skills from real work, while insights help teams understand quality, activity, history, and trends.

Memory Curator

Autonomously creates, recalls, updates, and consolidates project-scoped memory so stable decisions, conventions, and pitfalls do not need to be re-explained.

Skill Curator

Learns from completed work and improves the right reusable standalone or agent-owned skills while keeping those workflows visible and editable.

Grades & Project Pulse

Use grades, pulse summaries, and proactive analysis to surface repository health, quality issues, architecture risks, and actionable opportunities.

Reflection & History

Turn execution history into concise reflection summaries so teams can see what changed, what needs attention, and where repeated work is trending.

Usage Analytics

Understand task activity, token usage, cost, model breakdowns, and execution trends from project-scoped analytics rather than scattered provider consoles.

Reusable Workflows

Combine agents, skills, lifecycle hooks, schedules, task chains, channel triggers, and webhooks into repeatable automation managed inside the app.

Technology

Built on proven foundations

A modern, lightweight stack designed for reliability, performance, and simplicity. No Kubernetes required.

Go

Fast, compiled, concurrent. Single binary deployment with no runtime dependencies.

HTMX

Server-rendered hypermedia with real-time updates. No JavaScript framework overhead.

Templ

Type-safe HTML templating for Go. Compile-time checked components with excellent DX.

SQLite

Zero-config embedded database. WAL mode for concurrent reads. No separate database server.

Get Started

Up and running in minutes

With Go 1.26.5 or newer, clone the repository and let the startup script build and launch OpenVibely.

1

Clone

git clone https://github.com/openvibely/openvibely.git cd openvibely
2

Launch

./start.sh # Open http://localhost:3001
3

Set Up In The UI

# Add a model # Select or create a project # Start in Chat or Tasks
