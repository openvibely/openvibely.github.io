Core Platform

Everything you need to run AI-assisted engineering workflows

From task intake to PR-ready output, OpenVibely gives you structured automation with clear controls and real-time feedback.

Task Execution Workspace

Manage tasks across Backlog, Active, Scheduled, and Completed lanes. Follow execution in real time with streaming output and live code diff snapshots.

  • Drag-and-drop kanban workflow
  • Real-time execution and status streaming
  • Task thread follow-ups and context continuity
  • Live diff snapshots and execution history
  • Export/import as JSON

Smart Scheduling

Schedule one-time or recurring work with a visual calendar and automatic run triggering. Ideal for maintenance jobs, routine checks, and recurring automation.

  • Calendar-based scheduling UI
  • Once, interval, and recurring schedules
  • Automatic dispatch on due time

Model & Provider Flexibility

Use Anthropic, OpenAI, or Ollama with API key, OAuth, or CLI auth paths depending on provider. Configure models globally and set defaults per environment.

  • Anthropic, OpenAI, and Ollama support
  • OAuth, API key, and CLI auth options
  • Provider-aware fallback behavior

Worker Capacity Controls

Control throughput with global and per-project worker limits, queueing behavior, and live utilization metrics so automation stays predictable under load.

  • Global and per-project capacity limits
  • Queued/running visibility
  • Live worker utilization metrics

Git Worktrees & PR Flow

Run tasks in isolated git worktrees, review branch-level diffs, and create pull requests directly from task outputs. Keep parallel work streams clean and auditable.

  • Per-task git worktree isolation
  • Branch diff and merge controls
  • One-click pull request creation

Channels, Alerts & Observability

Operate through web UI, Slack, Telegram, and API with streaming events and alerting so teams can monitor and react without context switching.

  • Slack and Telegram bot workflows
  • GitHub channel integration options
  • Task, chat, and file-change SSE streams

Get Started

Up and running in minutes

Clone, configure, run. Start locally in minutes with no Docker or Kubernetes required.

1

Clone & Build

git clone https://github.com/openvibely/openvibely.git cd openvibely templ generate && go build -o bin/openvibely ./cmd/server
2

Configure

# Add your LLM provider credentials # Supports: Anthropic, OpenAI, Ollama cp .env.example .env # Edit .env with your API keys or OAuth settings
3

Launch

./start.sh # Dashboard available at http://localhost:8080
